FrontEnd

Technician is a person that will use mostly the frontend page. On that page, he has several very important functionalities like accepting or rejecting requests of the equipment, adding new equipment, editing already existing equipment and deleting equipment from the database. There is also a statistics page showing recent requests with the equipment name and group name, components condition (percentage of broken equipment), current project and most popular equipment with the equipment name and number of requests.

Django provides an admin page where those with permission can access and view the contents of the database, we thought that if there was any problem that couldn’t be resolved on the frontend, the technician could access the admin page and fix it there. The default implementation of the admin page provided by Django although functional, doesn’t provide features like finding an object for example. So we customize the admin page to make it easier to use, added features like filters and search bars, and make it display the information in a manner more akin to the excel sheets provided by the technician.

Mobile

The intent is to provide users (the students) with an easily accessible way to access information regarding the Makers Lab contents relevant to the users like the available equipment, their borrowed equipment, group details amongst others. As such we develop a web application to allow the students to access said contents.
