System Requirements and Architecture

System Requirements

This is the list of our system requirements, for users (students and/or teachers):

  • Create a new project
  • Add members to projects
  • Editing the contents of the project
  • Request and return equipment
  • See information about available equipment
  • See information regarding existing projects
  • See basic information about users
  • See statistics about the lab

Technician requirements

The requirements for the technician application is the following:

  • Stock management
  • User requests components through the wiki page
  • The program automatically updates the stock
  • When the user return the equipment, the technician validates (see if exists any problem) and the program update de stock
  • Everybody needs to know the contents of the stock, without any login

Non Functional Requirements

  • Usability: The goal is to create a webpage easy to use and navigate
  • Reliability: It is necessary for the system to work without any errors that would impact user experience.
  • Security: Due to the huge amount of users, it is important to make the system as secure as possible since it manages students projects and University equipment. Most of the security cases will be handled by the current University system.
  • Efficiency: Both the applications, and the webpage should be as lightweight as possible in order to allow the user to use them effectively regardless of the device they use.
  • Portability: It should be accessible by different web browsers and operate in the same way. Regarding the mobile application, it should be available to both Android and IOS.
