Authentication

The authentication to the web app follows the OAuth authentication provided by UA. That way, the students can enter the application with the standard login page that they would use to log into any other UA website, like elearning, paco, etc.

From the developers’ point of view, in order to do this, the proceedings documented here had to be put into practice. Basically there are 4 steps: request token, that will return a token; authorize, that will redirect the user, using the token previously requested, to an authorization page where the user can allow the processing of his/her data; access data, used to update the token to one that can be used to access the user’s data and get data, where the developer requests the user’s information. In this case, we chose to only access the user’s name and email, but if more information would be needed in possible future iterations, that could be done.
